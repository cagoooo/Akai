## 🔴 立即可做（本週）

### 1. 🔐 /admin 管理員驗證 (安全性 P0)
**核心任務**：目前的隱藏入口僅提供物理上的隱蔽。建議導入 Firebase Custom Claims，鎖定 API 級別的存取權限。

### 2. 🩺 工具鏈接存活監測 (健全性 P0)
**建議**：由於工具集包含許多外部連結，建議建立一個 Cron Job (使用 GitHub Actions 每週執行) 自動巡檢所有 `tools.url`。
- **實作**：使用簡單的 `fetch` HEAD request。
- **效果**：自動標記無效連結，避免使用者點擊到「404 頁面」，提升平台信譽。

### 3. 🧪 核心邏輯測試 (品質 P0)
**目標**：補足 `useAuth` 與 `useFirestore` 的 Mock 測試，確保版本迭代不損壞核心功能。

## 🟡 短期目標（1-2 週）— 體驗與深度互動

### 4. 🎨 個人化主題與節慶切換 (UX P1)
**概念**：允許使用者選擇自己喜愛的漸層配色，或在特定節慶（如教師節、聖誕節）自動切換限定版 UI 主題。

### 5. � 協作工具清單 (Playlist) (社群 P1)
**功能**：使用者可以建立自己的「工具清單」並生成專屬分享網址。
- **場景**：一線老師可以建立「三年級數學課專用工具包」直接分享給家長或同儕。

---

## 🟢 中期目標（1-4 個月）— AI 強化與個人化助手

### 6. 🔍 AI 語意搜尋 (Semantic Search)
**概念**：超越關鍵字匹配。使用者搜尋「我想教小朋友認識水果」，AI 能自動聯想到「遊戲類」或「繪本類」中相關的工具。
- **技術**：整合向量資料庫（如 Pinecone）或使用 Gemini Embedding。

### 7. 🤖 AI 教案產出系統
**概念**：輸入課程主題與目標級數，AI 結合平台上的工具，自動排版一份完整的 40 分鐘教案 PDF。

## 🔵 長期願景（3-6 個月）— 平台進化與生態系

### 8. 🎮 遊戲化班級營運 (Gamification)
**概念**：建立班級排行榜，學生使用工具累積的「經營點數」可以兌換虛擬獎勵或徽章。

### 9. 📱 跨端整合版本
**方案**：將 PWA 使用 Capacitor 打包成原生 iOS/Android App，進駐 App Store / Play Store 擴大接觸面。

---

## 🔧 持續演進的技術架構建議

1.  **動態載入優化**：針對非即時需要的組件（如後台與複雜圖表）使用 `React.lazy` 與 `Skeleton`，維持首頁載入在 1 秒內。
2.  **CDN 加速**：靜態預覽圖（PNG/WebP）移至 Cloudflare R2 或 Firebase Storage CDN，減輕主伺服器負擔。
3.  **i18n 多語言架構**：雖然目前以繁體中文為主，但建立好 `i18next` 架構有利於未來向國際教育市場推進。
